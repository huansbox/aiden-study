# PRD — 三下自然「期末」題庫練習

> 對齊來源：2026-06-09 grill-me session。設計細節見 `docs-dev/exam-final-pipeline-design.md`。
> 考卷來源與課綱界線見 `docs-dev/exam-paper-sourcing.md`。

## Problem Statement

小孩即將考三年級下學期自然「期末考」。現有的練習網站只涵蓋「期中」範圍（第1單元田園樂、第2單元溫度變化），無法用來複習期末範圍（動物、天氣）。家長手上已有期末考卷 PDF，但需要把它們變成小孩能在 pad 上點選作答的題庫；而且希望比期中站更好用——能針對「同一類概念」的題目集中練習，哪裡弱就練哪裡，而不是整個單元混在一起。

## Solution

沿用期中既有的「PDF 萃取 → AI 分類 → 靜態網站」流程，擴充出期末題庫，並合併進同一個網站：

- 網站最上層可切換「期中／期末」；期中內容改名保留、不移除。
- 期末分第3單元「動物」、第4單元「天氣」，每個單元下再分出數個「概念子主題（subtopic）」。
- 小孩除了能「整個單元」練習，也能點進某個 subtopic 只練那一類題目。
- 題目答案以官方答案卷為準；少數沒有答案卷的學校由 AI 補答案，並標記出來供家長人工複查。
- 第一階段只收「是非題」「選擇題」（最適合 pad 點選）；填一填、配合題暫不納入。

## User Stories

1. As a 家長, I want 把期末考卷 PDF 自動萃取成題目資料, so that 我不必手動逐題打字建題庫。
2. As a 家長, I want 萃取流程沿用期中那套, so that 我不必重學或重寫一套新工具。
3. As a 家長, I want 有官方答案卷的考卷以官方答案為準, so that 小孩練到的答案是正確的。
4. As a 家長, I want 沒有官方答案卷的題目由 AI 補答案並被標記出來, so that 我可以只針對這些題快速複查，而不必檢查全部。
5. As a 家長, I want 系統把超出期末範圍的題目自動標記排除, so that 小孩不會練到不考的內容。
6. As a 家長, I want 分類體系以桃子腳國小的考試範圍為基準, so that 練習內容貼近小孩實際會考的範圍。
7. As a 家長, I want 把含圖片、無法純文字作答的題目自動跳過, so that 題庫裡每題都能在 pad 上作答。
8. As a 家長, I want 重複出現的題目自動去重, so that 同一題不會一直重複出現。
9. As a 家長, I want 期末題庫合併進現有網站而不是另開一個站, so that 小孩只要記一個網址、一份進度。
10. As a 家長, I want 期中內容改名保留而非刪除, so that 之後若想複習期中還在。
11. As a 小孩, I want 在網站最上層選「期中」或「期末」, so that 我能快速找到現在要複習的範圍。
12. As a 小孩, I want 選一個單元做「全部練習」, so that 我能把整個單元的題目練過一輪。
13. As a 小孩, I want 「全部練習」答對的題移出、答錯的排到隊尾並可接續, so that 我能反覆練到全部答對為止。
14. As a 小孩, I want 選一個單元做「快速練習」抽 10 題, so that 我能在零碎時間快速複習。
15. As a 小孩, I want 快速練習優先抽我常錯、練得少的題, so that 我把時間花在最需要加強的題目。
16. As a 小孩, I want 點進某個 subtopic（如「動物分類」）只練那一類, so that 我能集中加強特定概念。
17. As a 小孩, I want subtopic 也能做「全部練習」和「快速練習」, so that 我在概念層級也有完整的練習方式。
18. As a 小孩, I want subtopic 的「全部練習」也能接續, so that 我中途離開後能從上次進度繼續。
19. As a 小孩, I want 做錯的題自動進錯題庫, so that 我之後能專門重練錯過的題。
20. As a 小孩, I want 錯題練習以單元為單位（第3單元錯題／第4單元錯題）, so that 我能針對整個單元的弱點複習。
21. As a 小孩, I want 錯題答對後從錯題庫移除、答錯留著排到隊尾, so that 錯題庫反映我目前真正還不會的題。
22. As a 小孩, I want 是非題用點 O／X 作答, so that 在 pad 上操作直覺。
23. As a 小孩, I want 選擇題用點選項作答, so that 在 pad 上操作直覺。
24. As a 小孩, I want 我的練習進度與錯題庫被記住, so that 換時間再打開還在。
25. As a 小孩, I want 期中的舊進度不會因為加了期末而被清掉, so that 我之前練的紀錄還在。
26. As a 家長, I want 期末題庫部署到同一個 GitHub Pages 網站, so that 小孩用同一個連結就能練。
27. As a 開發者, I want 萃取器是一個輸入文字、輸出結構化題目的純函式, so that 我能用考卷片段對它寫可重複的測試。
28. As a 開發者, I want 期中／期末的單元與子主題定義是可切換的設定, so that 我能用同一套程式處理不同學期而不改核心邏輯。
29. As a 開發者, I want 去重、答案合併、分類驗證是可獨立測試的純函式, so that 我能確信這些關鍵邏輯正確。
30. As a 家長, I want 舊課綱（108下／109下）的考卷被排除, so that 題庫只含與現行課本一致的內容。

## Implementation Decisions

**資料 pipeline（沿用期中三段式，擴充期末）**
- 萃取層維持「PDF 文字 → 結構化題目」的純函式介面；輸出欄位沿用期中：`source / section / number / text / options / answer / has_image`。
- 期末資料格式關鍵差異：題目卷括號為空（無內嵌答案）；答案卷與題目卷同版面但括號已填答案且含題目全文（等同期中「格式A 內嵌答案」）。因此**有答案卷的學校直接對答案卷萃取，一次取得題目＋官方答案**，不需另寫答案對齊模組。
- 萃取層需把期末出現的非目標大題標題（如「根據題意回答問題」「填一填」「配合題」）納入截斷規則，避免污染是非／選擇題。
- 分類層沿用「批次送 AI、回傳 unit/subtopic/confidence/answer」流程；把「單元與子主題定義（system prompt ＋ 合法子主題集合）」抽成**可切換設定**，期中一組、期末一組。
- 期末分類設定：`unit` = "3"（動物）／"4"（天氣）／"none"；subtopic 為下列八者之一或 "none"。
- 答案合併規則：有原始（官方）答案 → 用官方；無 → 用 AI 補，且該題標記為「需複查」。
- 範圍判定：以桃子腳國小涵蓋範圍為分類基準；不屬任何 subtopic 的題目歸 "none" 並排除於練習之外。
- 去重：以正規化後的題目文字為鍵去重（同期中）。
- 跳過：含圖片題、選項數不足的選擇題跳過（同期中）。

**期末 subtopic 分類體系（以桃子腳為基準）**
- 第3單元 動物：`動物分類`、`身體構造`、`生存與適應`、`觀察方法`。
- 第4單元 天氣：`風`、`氣溫測量`、`雨量降雨`、`天氣預報`。

**題庫資料**
- 最終題庫 schema 沿用期中：`id / unit / subtopic / type / text / options / answer / source`；`unit` 值域擴為 1–4（全域唯一）。
- 期中＋期末合併為單一題庫檔（簡單優先；單檔含 units 1–4）。此為預設決定，若載入或切換邏輯有困難可改分檔。

**網站（在現有單一 HTML 上改造）**
- 最上層新增「期中／期末」切換：期中顯示 units 1–2，期末顯示 units 3–4。
- 各單元卡片下列出其 subtopic 作為練習入口。
- 「全部練習」「快速練習」可作用於「整個單元」或「單一 subtopic」；「錯題練習」維持單元層級。
- 站名改為涵蓋期中＋期末（具體名稱待定，期中區塊標籤保留）。

**狀態儲存（localStorage，沿用既有 key，盡量不破壞期中進度）**
- 因 `unit` 3/4 全域唯一，挑戰佇列與錯題庫可直接新增 3/4，**期中既有進度無需遷移**。
- subtopic 層級的「全部練習」可接續：挑戰佇列改用「單元」或「單元＋subtopic」的複合鍵。
- 「快速練習」維持無狀態（不接續），直接以單元或 subtopic 過濾選題。
- 錯題庫項目維持「題目 id ＋ 單元」結構（錯題只到單元層）。

**深模組切分**
- 模組A 題目解析器（Python 純函式）：封裝雙欄偵測、section 切分、格式A/B 偵測、是非/選擇答案正規化、題目切分、選項抽取。介面：文字進、結構化題目出。
- 模組B 資料處理 helpers（Python 純函式）：去重、官方/AI 答案合併、unit/subtopic 合法性驗證。
- 模組C 學期分類設定（資料/設定）：期中與期末各一組單元、子主題定義，可切換。
- 分類器對 AI 的呼叫本身（非確定性、外部依賴）不屬深模組、不寫單元測試。

## Testing Decisions

**好測試的定義**：只驗證模組的外部行為（輸入 → 輸出），不綁內部實作細節；用真實考卷的文字片段當輸入，斷言萃取/處理結果，讓重構不會誤傷測試。

**要寫測試的模組**
- 模組A 題目解析器：用真實期末考卷片段建 fixture，涵蓋——
  - 雙欄排版的考卷不會把題目切斷；
  - 格式A（答案卷，括號內含答案）能同時正確抽出題目文字與答案；
  - 格式B（題目卷，空括號）只抽題目、答案留空；
  - 是非題 O／X 與各種變體正規化為 true／false；
  - 選擇題 ①–④／數字／全形 正規化為 "1"–"4"；
  - 選項正確切分；含圖片題被標記。
- 模組B 資料處理 helpers：
  - 重複題目文字被去重；
  - 有官方答案時不採用 AI 答案、無官方答案時採用 AI 答案且標記需複查；
  - 非法的 unit／subtopic 被擋下或歸 none。

**Prior art**：專案目前無自動化測試，這是第一批。測試以 Python（搭配 uv／pytest）撰寫，輸入取自 `pdfs_期末/` 真實考卷的代表性片段。

**不寫測試**：分類器的 AI 呼叫（非確定性）、網站 Picker/State（JS，專案無 JS 測試環境，對單人維護過重，以手動測試為主）。

## Out of Scope

- 填一填、配合題的萃取與作答（約佔每份考卷 44%）；未來可能用 AI 轉成選擇題後納入，本次不做。
- 含圖片才能作答的題目。
- 舊課綱（108下／109下）考卷，以及興南、安和的 108下 期末卷。
- 其他出版社、其他縣市的考卷（本次鎖定康軒、臺北＋新北，且分類以桃子腳為基準）。
- 網站 Picker/State 的自動化測試與相應的 JS 測試環境建置。
- 期中既有題庫內容的重新分類或調整。
