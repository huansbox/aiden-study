## 為什麼（rationale）

期中題庫（單元1 田園樂、單元2 溫度變化）已上線。小孩即將考三下**期末**，需「比照辦理」把期末考卷做成 pad 練習題庫，並**比期中更細**：能按「類似概念」分群練習（期中站只能按單元練）。

## subtopic 分類體系（以桃子腳為準）

**第3單元 動物**
- `動物分類`：哺乳/爬蟲/兩生/鳥/魚類的辨識與特徵
- `身體構造`：頭軀幹四肢、翅膀羽毛鱗片鰭蹼爪，及構造對應的功能
- `生存與適應`：保護色/警戒色、瞳孔受光變化、食性、棲地（如蚯蚓）
- `觀察方法`：戶外觀察動物的方法與正確態度

**第4單元 天氣**
- `風`：風向、風力、觀測工具（風向標／風向風力計）
- `氣溫測量`：氣溫計、百葉箱
- `雨量降雨`：雨量測量（容器）、西北雨／太陽雨成因
- `天氣預報`：預報種類（今日/一週/國際都市/漁業）、資訊判讀、降雨機率意義

> 不屬以上任一 subtopic 的題目 → `unit:"none"`，比照期中排除。

## 核心機制：考卷資料格式（關鍵發現）

- **期末題目卷**：括號是**空白**的（是非 `（ ）1.`、選擇 `( )1.`），只有題目文字，**無內嵌答案**。
- **期末答案卷**：與題目卷**版面完全相同，但括號內已填答案**（是非 `（O）1.`、選擇 `（2）1.`），且**題目全文也都在**。等同期中的「格式A 內嵌答案」。
- 推論：**有答案卷的學校，直接對「答案卷」跑現有 `extract.py`（格式A），一次取得「題目＋官方答案」**，不需另寫答案對齊邏輯。
- 題型：是非／選擇／填一填／配合題四種。現行 `extract.py` 的 `NON_TARGET_SECTION` 會在填一填/配合題 header 截斷 → 天然丟棄（符合決策3）。**需補**：把桃子腳用的 section header「根據題意回答問題」「填一填」「配合題」確實納入 `NON_TARGET_SECTION`，避免污染。

## 完整流程

```
pdfs_期末/ (10 份現行課綱期末，題目卷＋部分答案卷)
   │
   ├─[有答案卷] 安和×4 + 桃子腳×2
   │     └ extract.py 跑「答案卷」 → 題目＋官方答案（格式A，是非+選擇）
   │
   └─[無答案卷] 民權×4
         └ extract.py 跑「題目卷」 → 只有題目（空括號）
   │
   ▼
raw_questions_期末.json  (含 source, section, number, text, options, answer?)
   │
   ▼ classify.py（新 system prompt：期末 taxonomy）
   │   - unit: "3"/"4"/"none"
   │   - subtopic: 上述 8 個之一 或 "none"
   │   - 民權無答案者：AI 補 answer + 低信心/AI補 → 標記複查
   │   - 去重（normalize text）、比照期中
   ▼
classified_questions_期末.json
   │ （fix_classified 手動修正、skipped 清單）
   ▼
docs/questions_期末.json  (schema 同期中 + unit=3/4)
   ▼
docs/index.html（改造：期中/期末切換 + subtopic 練習）
```

## 實作骨架（檔案層級）

**資料處理**
- `scripts/extract.py`：加 `--target` 或輸出路徑參數，支援跑期末來源（答案卷／題目卷）；補 `NON_TARGET_SECTION`（根據題意/填一填/配合題）。輸出 `data/raw_questions_期末.json`。
- `scripts/classify.py`：抽出 `SYSTEM_PROMPT` 與 `VALID_SUBTOPICS` 為可切換設定（期中 vs 期末）；期末用第3/4單元 taxonomy。輸出 `data/classified_questions_期末.json`。
- 合併：期中＋期末 → `docs/questions.json`（單一檔，`unit` 1–4 全域唯一）或分檔 `questions.json` + `questions_期末.json`（**尚待決定**，見下）。

**網站 `docs/index.html`**
- 最上層加「期中／期末」切換（期中＝units 1,2；期末＝units 3,4）。
- 各單元卡片下，列出該單元的 subtopic 作為練習入口；「全部練習」「快速練習」可作用在 subtopic。
- 站名改名涵蓋兩者（如「三下自然題庫」），期中區塊保留。

**localStorage（key `aiden_study_v2`，盡量不破壞期中進度）**
- `unit` 3/4 全域唯一 → `challenge`、`errorBank` 直接新增 3/4，**期中進度不需遷移**。
- subtopic 級「全部練習」可續：`challenge` 改用複合 key，如 `"3"`（整單元）、`"3/動物分類"`（subtopic）。
- 「快速練習」無狀態（不續），可直接吃 subtopic 過濾。
- `errorBank` 項目維持 `{questionId, unit}`（錯題只到單元層）。
