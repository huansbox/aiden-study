注音 TTS 可行性測試

在 iPad Safari 打開，逐條點按來聽。目的：決定注音學習 app 的聲音走「家長親錄」還是「Apple 語音生成」。

0. 語音設定

（尚未發聲）

1. 單獨注音符號（直接餵符號）

聽重點：ㄅ 應該唸「ㄅㄛ」。若唸成英文字母、拼字、或沉默 → 這條路對「單獨符號」不可行。

2. 符號的載體字（教學唸名用真實字代替）

對照組：如果第 1 區失敗，改唸這些字能不能當「符號的聲音」？（玻=ㄅ、坡=ㄆ、摸=ㄇ、佛=ㄈ、阿=ㄚ、屋=ㄨ；注意「佛」是二聲，教學上 ㄈ 唸一聲的「ㄈㄛ」，聽聽差多少）

3. 拼音組合（直接餵注音＋聲調符號）

聽重點：①有沒有唸成正確音節＋正確聲調 ②「ㄅㄨˊ」這種沒有常用國字的組合行不行。任何一條唸錯或拆開唸 → 原始注音字串這條路不可行。

4. 真實字載體（拼音用同音國字代替）

對照組：用「八拔把爸」代替「ㄅㄚ 四聲」。聽重點：單字唸得標不標準、聲調對不對（特別是三聲的「把、馬、斧」單獨唸會不會變怪）。

5. 詞彙（自然度參考）

聽重點：整體音質。小孩要聽幾百次，這個聲音你能接受嗎？

判讀表

✅ 第 3 區全對 → 可走「Mac 批次生成音檔」，維護最省力
✅ 第 3 區失敗、第 4 區可以 → 走「真實字載體生成」，但無字組合（如 ㄅㄨˊ）要親錄補洞
✅ 第 1、2 區都不理想 → 至少「符號單音」要家長親錄
❌ 音質整體聽不下去 → 全部親錄（約 50 段，我做錄音工具頁）
