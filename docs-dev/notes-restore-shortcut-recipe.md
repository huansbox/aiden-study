# 「從備忘錄還原」捷徑配方（iOS／macOS 捷徑 App）

> 父 PRD：[#7](https://github.com/huansbox/aiden-study/issues/7)／本票：[#10](https://github.com/huansbox/aiden-study/issues/10)
>
> 這份是**真相源**。捷徑本體存在 iCloud、不進 git；哪天要重建，照這份逐步抄即可。
> 介面標籤以**繁體中文**捷徑 App 為準，括號內附英文。

## 這顆捷徑在還原流程的位置

還原路徑只有「讀備忘錄、列日期清單讓家長挑」這一段非捷徑不可——iOS 沒有任何網頁 API 能讀備忘錄。其餘（解碼、驗證、二次確認、覆蓋、重載）全在網站側完成。

```
網站「從備忘錄還原」鈕
  → shortcuts://run-shortcut?name=Aiden還原   （只喚起捷徑，不帶資料）
    → 捷徑：尋找備忘錄 → 從列表選擇 → 取內文 → base64 → 打開 URL
      → https://huansbox.github.io/aiden-study/#restore=<base64>
        → 網站讀 hash → 解碼＋驗證 → 二次確認（顯示日期）→ 覆蓋 localStorage → 重載
```

捷徑是**自給自足**的：它自己去翻備忘錄、自己讓家長挑，網站的深連結**不傳任何資料**進來（見 `docs/index.html` 的 `buildShortcutDeepLink`，只帶捷徑名稱）。

## ⚠️ 耦合點：捷徑名稱必須一字不差 = `Aiden還原`

網站深連結把捷徑名稱**寫死**在 code：

- `docs/index.html` → `const SHORTCUT_NAME = "Aiden還原";`（約 line 1093）
- 喚起的 URL ＝ `shortcuts://run-shortcut?name=Aiden%E9%82%84%E5%8E%9F`

捷徑名稱差一個字（含全形／半形、大小寫、空白），網站的「從備忘錄還原」鈕就喚不起它。改名要同步改 code 的 `SHORTCUT_NAME`。

## 備份筆記長什麼樣（捷徑要還原回去的格式）

家長在網站按「備份到備忘錄」→ 系統分享選單 → 點「備忘錄」，存出來的筆記內容是：

```
AIDEN備份 2026-06-15 14:30
{"mastered":{...},"challenge":{...},"errorBank":[...],"stats":{...},"flagged":[...]}
```

- **第一行**＝記號 `AIDEN備份` ＋空格 ＋ 日期時間（`YYYY-MM-DD HH:MM`）。備忘錄以首行為標題，所以筆記標題就帶日期，家長掃一眼就知哪份最新。
- **第二行起**＝原始 JSON（整份 localStorage 存檔）。
- 格式契約寫在 `docs/index.html` 的 `<backup-pure>…</backup-pure>` 區塊；改那裡務必連 `tests/test_backup_pure.mjs` 一起跑。

捷徑要做的，就是把這整則筆記內文（**含**第一行記號）base64 編碼後接到 `#restore=` 後面。

# 逐步建捷徑（macOS）

## 前置：建立並命名捷徑

1. 打開 **「捷徑」App**（Spotlight 打「捷徑」或 Launchpad 找）。
2. 按鍵盤 **⌘N**（或視窗左上角工具列的 **＋** → 「建立新捷徑」）。會跳出一個**空白捷徑編輯器**視窗。
3. **命名**：編輯器最上方中央會看到捷徑名稱（預設叫「捷徑」之類），**點它一下** → 跳出可改名稱／圖示的小視窗 → 把名稱打成 **`Aiden還原`**（一字不差）→ 點空白處收起。
   - 若點不到：先關掉這個編輯器，回捷徑列表，在這張捷徑格子上**按右鍵 →「重新命名」**。

## 通用操作：怎麼加一個動作（學這一次，下面 7 個動作都這樣做）

- 看編輯器**右側**的「動作資料庫」(Action Library)，最上面有個**搜尋框**。
  - 沒看到右側面板？點編輯器**右上角**像方塊／積木的圖示，或選單列 **顯示方式 → 動作資料庫**，把它叫出來。
- 在搜尋框**打動作名稱** → 結果列表出現該動作 → 在那個動作上**按兩下**（double-click），它就會接到流程**最下面**。
- 動作裡**藍色／可點的字**就是要設定的參數，逐個點開設定即可。
- 想刪錯加的動作：點該動作 → 按 **Delete**。

> 重點觀念：動作由上往下依序執行，**下一個動作預設吃上一個動作的結果**。所以照順序加就會自動串好，多數時候不用手動接變數。

## 動作 1／7：尋找備忘錄（Find Notes）

1. 搜尋框打 **`尋找備忘錄`** → 按兩下加入。加進來後預設顯示「**尋找所有備忘錄**」。
2. 加篩選條件：點動作裡的 **「加入篩選條件」**（或動作上的一個 **＋**）→ 出現一條篩選列，分三格。
3. 從左到右設定這三格：
   - 第 1 格（屬性）：點開選 **「名稱」**(Name)
   - 第 2 格（條件）：點開選 **「包含」**(contains)
   - 第 3 格（值）：點一下，打 **`AIDEN備份`**
4. （可選，讓最新的排最前面）動作下方若有「**排序方式**」：選 **「上次編輯日期」**，順序選 **「最新到最舊」**。不設也能動，只是清單順序差別。

完成後這個動作大意是：「尋找名稱包含 AIDEN備份 的所有備忘錄」。

> 若你的版本看不到「加入篩選條件」：先按兩下加入「尋找備忘錄」，動作標題那行通常有「**符合 [全部]**」字樣，旁邊就有 ＋ 可加條件。找不到就回報給我，我們換一種找法。

## 動作 2／7：從列表選擇（Choose from List）

1. 搜尋框打 **`從列表選擇`**（或 `從清單中選擇`）→ 按兩下加入。
2. 它的輸入會**自動接上一步**，顯示成「從『備忘錄』中選擇」。不用動。
3. （可選）動作裡「提示」欄可打 **`選一份備份還原`**，執行時會顯示這句。

執行時這一步會跳出清單，每列顯示備忘錄標題＝`AIDEN備份 2026-06-15 14:30`，家長就靠**日期**挑一份。

## 動作 3／7：取得備忘錄內文（Get Details of Notes → 內文）

1. 搜尋框打 **`取得備忘錄詳細資訊`** → 按兩下加入。
2. 輸入自動接「**所選項目**」（上一步選中的那則）。
3. 把動作裡要取的細項點開，選 **「內文」**(Body)。

> 若搜不到這個動作：可**略過本步**，直接進動作 4，在 Base64 編碼時把「所選項目」當輸入，捷徑通常會自動取其內文。顯式取「內文」只是更穩。

## 動作 4／7：Base64 編碼（Base64 Encode）

1. 搜尋框打 **`Base64 編碼`** → 按兩下加入。輸入自動接上一步的內文。
2. 點開動作裡的選項，把 **「換行」(Line Breaks)** 設成 **「無」(None)**。
   - ⚠️ 一定要選「無」，否則編碼結果夾雜換行會塞壞網址。

## 動作 5／7：三個「取代文字」轉成網址安全格式

標準 base64 含 `+` `/` `=`，放進網址可能被瀏覽器改寫而還原失敗。連加**三個「取代文字」**轉成網址安全版（與網站 `encodeBackup` 同一套）。

⚠️ **Mac 的欄位順序是反的**：動作顯示成「在 [輸入] 中**以 A 來取代 B**」＝把 B 換成 A。所以：**「取代 ___」放要找的舊字元、「以 ___」放換成的新字元**。三個動作的目標：

| 要找的（填「取代 ___」）| 換成（填「以 ___」）|
|---|---|
| `+` | `-` |
| `/` | `_` |
| `=` | （留空，什麼都不打）|

逐步（搜尋 `取代文字` → 按兩下加入，做三次）：
1. 「以 ___」打 **`-`**、「取代 ___」打 **`+`**。
2. 「以 ___」打 **`_`**、「取代 ___」打 **`/`**。
3. 「以 ___」**留空（不要打字）**、「取代 ___」打 **`=`**。

注意事項：
- 第 3 個的「以 ___」**一定要空的**。若那格顯示淺色「世界」之類字樣，那是 Apple 的空欄位提示＝**你沒填**＝正確；只要不是你真的打了字就好。
- 三個的先後順序互不影響（`+`/`/`/`=` 各自獨立），不用刻意調。
- 三個動作裡的「**正規表達式**」「**不分大小寫**」維持**關閉**（預設就是關）。
- 每個「取代文字」的**輸入要接前一個**的結果。捷徑通常自動接最近的變數，確認輸入顯示「**已更新的文字**」（第一個接「**已 Base64 編碼**」）即可。

## 動作 6／7：文字（Text）— 組出還原網址

1. 搜尋框打 **`文字`**(Text) → 按兩下加入。
2. 在文字框打：`https://huansbox.github.io/aiden-study/#restore=`
3. 游標**停在最後面**，按右鍵（或文字框上方的變數列）選 **「插入變數」** → 選**最後一個**「**更新後的文字**」（也就是動作 5 第三個取代文字的結果）。
   - 結果像：`https://huansbox.github.io/aiden-study/#restore=` 後面緊跟一個藍色變數膠囊。

## 動作 7／7：打開 URL（Open URLs）

1. 搜尋框打 **`打開 URL`** → 按兩下加入。輸入自動接上一步「文字」。
2. 不用其他設定。

## 收尾

- 捷徑 App 會**自動存檔**，直接關視窗即可。再確認一次名稱是 **`Aiden還原`**。
- 順手把這顆捷徑「**分享 → 拷貝 iCloud 連結**」貼到自己的備忘錄留底，誤刪可一鍵重灌。

> 設計上不需要 URL-safe 之外的容錯：`decodeBackup` 對標準與 URL-safe base64 都能解（已用 node 驗證）。但 `+`/`/`/`=` 在真機網址上的行為留待 #11 校準，**請照上面 URL-safe 版（含動作 5 三個取代）做**最保險。

## Mac 驗證（建好捷徑後跑）

備好 helper：[`scripts/seed_test_backup_note.sh`](../scripts/seed_test_backup_note.sh)。它用 osascript 在「備忘錄」建一則測試備份筆記，再印出後續步驟。

```bash
bash scripts/seed_test_backup_note.sh
# → 在「備忘錄」出現一則「AIDEN備份 <今天日期>」測試筆記
shortcuts run Aiden還原
# → 跳出清單 → 選那則 → 自動在預設瀏覽器開 #restore=… → 二次確認 → 還原
```

驗收要看到：捷徑**找到**測試筆記、清單**列出日期**、開站後網址帶**正確的 `#restore=`**、二次確認顯示**對應日期**、按確定後 localStorage 被覆蓋（mastered 變成測試筆記裡那份）。

> ⚠️ osascript 寫進備忘錄的內文是經 HTML 轉換的，跟「真的從網站分享到備忘錄」未必逐字相同。osascript 版當**快速冒煙測試**；最終驗收請**也**用真機從 live 站按「備份到備忘錄」存一則真備份，再跑捷徑還原一次，這才跟正式流程一致。

## 同步到 iPad

- 同一 Apple ID，且「捷徑」與「備忘錄」的 iCloud 同步都開啟（已確認）。
- 在 Mac 建好 `Aiden還原` 後，iCloud 會把它同步到 iPad 的捷徑 App（可能要等幾分鐘／開一下 iPad 捷徑 App）。

## 後備路徑（捷徑壞掉也救得回）

捷徑沒裝／壞掉時，iOS 不會回呼網站，網站偵測不到，所以「從備忘錄還原」鈕旁邊直接給了後備指引：

- 打開備忘錄那則備份 → 全選內文複製 → 網站「進度備份/還原 → 匯入進度（貼文字框）」貼上 → 還原。
- `parseBackup` 容忍**帶或不帶**第一行記號，所以整則貼或只貼 JSON 都能還原。

## 待 #11 真機校準

- 用**最大**存檔在真機實測：`navigator.share` 文字大小、`#restore=` 網址長度上限。
- 必要時調 `docs/index.html` 的 `BACKUP_SIZE_WARN`（目前 40000 字元，保守值）。
- 觀察 standalone（加到主畫面）模式 localStorage 是否真被 iOS 約 7 天清除（非阻擋觀察項）。
