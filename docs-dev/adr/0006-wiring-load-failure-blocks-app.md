# 接線層腳本沒載到＝拒絕開站，取代「可練不持久」靜默降級

脈絡：#40-B 之前，共用腳本（sync-v1.js）載入失敗時 app 靠各自內嵌的後備碼硬撐（可練不持久、定錨後備、身分守衛）——這批後備碼是四 app 複製債的一部分，且 #40-A 那類 bug（跨 child 寫入、匯入定錨遺失）全部出自「腳本沒載到但 app 繼續跑」的半殘狀態。接線層抽進 `shared/wiring-v1.js` 後，後備碼的家在共用檔裡，「共用檔本身沒載到」時 app 端已無憑依。

決定（2026-07-17）：`wiring-v1.js` 沒載到 → app 顯示「載入不完整，請重新整理頁面」並拒絕任何存檔讀寫；app 端只留幾行偵測碼。行為變更點：不帶 `?child=` 的舊網址（含備忘錄還原捷徑轉送）過去遇腳本載失敗仍可練，改後吃一次錯誤頁——載入失敗是暫時性的，重整即癒，`#restore=` 仍在網址上、流程不斷。sync-v1.js 沒載到（wiring 有載到）的行為維持原樣：指名 child 拒開站、legacy 網址可練不持久＋定錨後備——這套語意單一副本住在 wiring 裡，不再是四份複製。

取捨：用「罕見情況下多按一次重新整理」換掉「四份最容易長 bug 的沉默降級後備碼」。
