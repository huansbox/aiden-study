# 進度身分＝family token＋child 維度，不做登入帳號

脈絡：進度要跟人走（哥哥弟弟各自的進度、跨裝置、抗 iOS 約 7 天清 storage），直覺方案是「每個小孩一個登入帳號」。

決定：不做登入帳號。同步服務以**家庭密鑰**（隨機 token，藏於主畫面圖示網址參數）驗身，雲端進度 key＝`{child}:{app}`；「現在是誰在用」由 hub 選人畫面（或各裝置圖示帶死參數）決定，不靠密碼。

為什麼：5 歲與 9 歲使用者面前，任何登入牆都是使用障礙；單一家庭不需要帳號系統的後端、密碼管理與復原流程——token＋child 維度已完整交付「進度跟人走、可跨裝置」。此設計沿自注音 PRD 經三 reviewer 審查的同步方案，本平台推廣為全站規約。
