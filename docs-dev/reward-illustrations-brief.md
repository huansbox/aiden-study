2. 一組 **8 張**「通用慶祝/鼓勵」插畫（無特定主題，給錯題清除＋缺圖時用）。
3. 一份 **`docs/shared/rewards.json`** manifest（格式見下），把檔名填進去。

圖檔放 `docs/shared/rewards/`。

【我的回答】原本「約 136 張 subtopic」建議改成「128 張 subtopic + 8 張通用鼓勵 = 136 張總量」。理由：下表各 subtopic 建議張數加總是 128；136 是把通用鼓勵池也算進去後比較合理的第一版總量。若之後想縮小第一波，也可以先做高題量 subtopic，但完整第一版我建議直接做 136 張。

## 用途與調性
給小學三年級小孩在 pad 上做題，**每做完一批/通關跳一張當獎勵**。調性：童趣、明亮、正向、單一主體清楚。可參考 `stroke.gh.miniasp.com` 那種乾淨童趣感。

## 規格
- **尺寸**：**交付物＝1024×1024 WebP（硬驗收）**。master 1536×1536 為理想來源規格、**非硬性**——imagegen 原生 ~1024 即可，但**勿從更小圖（如 512）上採樣**（會在約 400px 顯示框糊掉）。
- **構圖**：**1:1 正方形、滿版實底（自帶背景）**。`【我的回答】採用實底滿版方形，不採透明去背。` 理由：iPad 上獎勵圖會是主視覺，實底比較像收藏卡、完成感更強；透明去背在大量 AI 批次時品質不穩，也會讓程式側容器、陰影、背景搭配變複雜。
- **檔案**：WebP，lossy q80–85（平塗可順手比 lossless 哪個更小更利），**單張 ≤ ~100KB**。
- **master 不要 commit 進 `docs/`**（1536 PNG 太大）→ gitignore 或放你工作區，`docs/shared/rewards/` 只放 1024 WebP。
- **檔名自由**（靠 manifest 對應），建議 `<主題英文>_<nn>.webp`，如 `wind_01.webp`。

## 設計圖片 session 回答（我的回答）

### 1. 背景與視覺形式
採 **實底滿版方形插畫**。每張圖自帶乾淨背景，不需要透明 alpha。程式側可以把它當一般 1:1 圖片置中顯示，不需要再替不同圖補底色。

### 2. iPad 呈現尺寸
交付 1024×1024 WebP（原生/接近原生產出、勿上採樣）；master 1536 為理想非必須。1024 在 iPad 直式顯示約 400px 已足夠清楚，也能控制下載量。構圖要保留主體在中間偏大，避免把關鍵細節放在四角。

### 3. 圖片內容邊界
每張圖要扣 subtopic，但不要做成「題目解答圖」。可以出現具體物件與生活場景，例如小數用量尺/水瓶/價格牌，溫度用冰塊/巧克力/溫度計，綠色消費用環保袋/回收/環保標章概念。避免可讀文字、數字算式、題目句子、商標、真實品牌。

### 4. 風格方向
沿用目前試做圖：明亮、童趣、平塗插畫、單一主體清楚、細節適中。iPad 比手機可容納稍多背景道具，但每張仍要一眼看懂主題，不做雜亂場景。

### 5. 張數決定
接受程式側的 `clamp(ceil(題數/20), 3, 8)`，第一版建議固定為：
- subtopic 主題圖：128 張。
- `__generic__` 通用鼓勵圖：8 張。
- 合計：136 張。

理由：這個數量比線性每 5 題一張更務實，搭配 shuffle-bag 已能避免短期重複；8 張通用鼓勵圖也足夠支援錯題清除與缺圖 fallback。

### 6. 檔名與 manifest
manifest key 使用題庫 subtopic 原字串；檔名建議用 ASCII slug + 兩位數流水號，例如 `wind_01.webp`、`green_consumption_03.webp`。檔名不承載語意也可以，但要完整列入 `docs/shared/rewards.json`。

### 7. 可接受的漸進交付
完整第一版目標是 136 張。不過若需要先驗證程式流程，可先交付：
- 所有 `__generic__` 8 張。
- 每科各 1–2 個高題量 subtopic。
- 其餘 subtopic 之後補上。

理由：程式側已有 roll-up / fallback 設計，缺圖不會卡作答；但正式給小孩使用前，最好補齊全部 subtopic，主題關聯才穩定。

## manifest 格式（填好交回）
`docs/shared/rewards.json`：
```json
{
  "pools": {
    "風": ["wind_01.webp", "wind_02.webp", "wind_03.webp"],
    "天氣預報": ["forecast_01.webp", "forecast_02.webp", "...共 6 張..."],
    "綠色消費": ["green_01.webp", "green_02.webp", "green_03.webp"],
    "__generic__": ["cheer_01.webp", "...共 8 張..."]
  }
}
```
- key＝上表 subtopic 字串（逐字）＋ `__generic__`；value＝`assets/rewards/` 下的檔名陣列。
- **可漸進交付**：一個主題補一個、上線一個都行；程式側對缺的主題會自動往上退（不顯示也不出錯），不會卡。
