## 一、整條 pipeline（怎麼從 PDF 到網站）

```
pdfs_期末/*.pdf
  │  uv run python scripts/extract.py --input <檔1> <檔2> ... --output data/raw_questions_期末.json
  ▼  （extract.py：純函式 parse_questions_from_text；雙欄偵測 is_true_two_column；NON_TARGET 截斷）
data/raw_questions_期末.json
  │  uv run python scripts/classify.py --semester final
  ▼  （claude -p sonnet 批次分類 unit3/4 + 8 subtopic；無官方答案者 AI 補 + needs_review；去重/驗證走 data_helpers）
data/classified_questions_期末.json
  │  uv run python scripts/build_review_list.py   → review_期末_ai答案.md（AI 答案複查清單）
  │  uv run python scripts/build_questions.py      → 合併進 docs/study/questions.json（含 id 去碰撞、空選項過濾）
  ▼
docs/study/questions.json（期中602 + 期末；網站讀取）
  ▼  docs/study/index.html（期中/期末切換 + subtopic 練習，預設停期末）
```

期中同一套：`extract.py`（無參數＝讀 docs/，但期中 PDF 在 `pdfs/`，用 `--input pdfs`）＋`classify.py`（無 `--semester`＝mid）。

## 二、最容易踩的坑（務必先讀）

1. **雙欄污染**：答案卷/題目卷常是雙欄。`extract.py` 對每頁判斷是否雙欄（`is_true_two_column`）。
   - 安和答案卷第 2 頁右欄是「配合題答案欄」（`（D）(1) 25%` 型），曾被判為非雙欄→整頁逐行交錯，污染選擇題選項。已加偵測（matching_ans pattern）修掉。
   - 殘留：頁首/頁尾（「第 N 頁」「第 2 次定期評量…」）偶爾混進題目尾巴或選項。桃子腳112 mu11 的題幹甚至被截成「確？」、選項④混入頁尾——**這類「題幹/選項被雙欄打散」是目前最大的資料品質弱點**，靠 review 抓。
2. **NON_TARGET 截斷**：`NON_TARGET_SECTION` regex 界定「選擇/是非大題」的結尾。期末新增了 `填一填`、`根據題[目意]`。改這條 regex **會影響期中**（桃子腳112 期中也有「根據題意」），改完務必跑 `uv run pytest`（`tests/test_regression.py` 比對期中 golden `data/raw_questions.json`）。
3. **答案卷 ≠ 都能直接抽官方答案**：
   - 安和×4：格式A（括號內已填答案）→ 直接抽到官方答案。
   - 桃子腳112 答案卷：答案以**紅字標在選項文字上**、括號是空的 → 抽不到。改用「題目卷 + AI 補答案」。
   - 桃子腳110 答案卷：**純掃描圖片**（chars=0）→ 抽不到，改用題目卷 + AI 補答案。
   - 偵測圖片 PDF：`pdfplumber` 該頁 `extract_text()` 回空字串、`page.images` 有東西。
4. **claude -p 批次逾時**：sonnet 一批 30 題偶爾逾時（3 retry 全失敗→整批標 none，會「假性 none」吃掉題目）。已把期末 batch 降到 15、timeout 300s。重跑後若看到 `classify_reason=="分類失敗"` 的 none，就是逾時造成，要重跑。
5. **id 去碰撞**：考卷內偶有重複題號（安和110 MC 有兩個 18）。`build_questions.py` 的 `unique_id` 會對撞號加 `-2`，否則網站 `questionMap`（id→題）會默默吃掉一題。
6. **空選項 MC**：圖片型選擇題（選項是圖示、文字空白）會有「4 個空字串選項」。過濾條件是**有效（非空）選項 < 2**，不是 `len(options) < 2`。
7. **比對 reviewer 結果時 k↔題目要對齊**：review 用 `k = f"{source_stem}#{section[:2]}{number}"`。我曾把截斷顯示的 q 文字跟錯 k 配對，導致分歧報告標錯題號——**比對一律從 classified 的完整資料用 k 去查，不要用截斷過的暫存欄位**。

## 三、可切換學期設定（模組C）

`scripts/classify.py` 的 `SEMESTERS` dict：`mid` / `final` 各一份 config（system_prompt、valid_units、valid_subtopics、model、batch_size、IO 路徑）。核心分類邏輯與學期無關。要加新學期＝加一筆 config。**期中 prompt 一字不能動**（regression）。

純函式測試：`tests/test_parser.py`（parser）、`tests/test_data_helpers.py`（去重/答案合併/驗證）、`tests/test_classify_config.py`（config）、`tests/test_regression.py`（期中萃取 golden，PDF 不在時自動 skip）。`uv run pytest`。

## 四、AI 答案的可信度與複查

- 哪些是官方 vs AI：classified 的 `needs_review`（true=AI 補）；或看 `source`（安和_答案=官方，民權/桃子腳_題目=AI）。`docs/study/questions.json` **不帶** needs_review，只能靠 source 區分。
- `review_期末_ai答案.md`：所有進題庫的 AI 答案題清單，供人工核對。
- **單票盲審（review A）**：用 workflow 開 N 個 agent，各讀 `data/_review_blind.json`（去掉答案的盲版）盲答，與記錄比對只回分歧。123 題只 4 分歧（96.7% 一致），其中 2 題其實 AI 對、1 題 AI 可能錯（桃子腳110 mu16 漁業氣象，海水溫度才對）、1 題題幹壞掉。結論：sonnet 補答案整體可信，弱點在「題目被雙欄打散」而非答案判斷。
  - workflow 傳大量資料給 agent 的解法：**讓 workflow agent 自己用 Read 讀檔**（已驗證可讀專案檔），args 只傳小東西（如 chunk 範圍）；不要把幾十 KB 塞進 args（巢狀陣列還會傳成非陣列導致 `.map` 爆掉）。

## 五、網站（docs/study/index.html，單檔零依賴）

- localStorage key `aiden_study_v2`：`{challenge, errorBank, stats, semester}`。
- `challenge` 用**複合鍵**：整單元 `"3"`、subtopic `"3/動物分類"`（`challengeKey(unit, subtopic)`）。
- 期中/期末切換：`SEMESTERS`（mid=units1,2 / final=units3,4，含 subtopics）；`state.semester` 記住選擇，**預設 final**。
- 錯題模式 `quiz.unit` 是**單元陣列**（`[3]` 或 `[3,4]`），`getErrorBank` 支援陣列；錯題維持單元層級（不分 subtopic）。
- PWA：`apple-mobile-web-app-capable` + `manifest.json`（display standalone）→ iPad 可加到主畫面當 app；無 icon（iOS 用頁面快照）。
- 改完務必本機 `uv run python -m http.server` + 瀏覽器（或 Playwright）手動測；JS 無自動化測試。

## 七、第二批：其他縣市擴充（2026-06-10）

目標：找其他縣市同版本（康軒）同範圍（三下自然期末）現行課綱（110下+）卷擴充，官方答案優先、否則 AI 補＋複審，只做是非＋選擇。

### 來源擴充
- 第一批只查臺北＋新北；用 `scripts/sweep_tcool.ps1`（**空 city＋空 period**）一次掃全台所有縣市分頁 → `data/tcool_sweep_all.json`（108 筆）。tcool 全台 20 縣市。
- 全台·康軒·三下自然·期末·110下+ 共 25 筆，扣掉第一批已收，**新發現 15 份其他縣市卷**（臺中/彰化/臺南/高雄/基隆）。

### 本批實作（官方答案 8 份中可用的）
家長決策：分兩階段（官方先做、AI 後做）；圖片答案卷走視覺讀官方。實際只有 4 份乾淨可做：
- **三民111**：答案卷格式A 文字直抽（最可靠）。但雙欄打散，是非 12 題只抽到 6。
- **四維112、廣興111、廣興112**：題目卷文字可抽、答案卷掃描圖片 → 視覺讀官方答案注入。
- 流程與工具見 `docs-dev/exam-paper-sourcing.md`「圖片型官方答案卷的視覺判讀流程」。

### 新踩的坑（parser）
1. **PUA 選項標記**：彰化廣興111 題目卷的 ①②③④ 是私用區碼位 **U+F06A–F06E**（某字體），pdfplumber 抽出 PUA 碼點 → 選項偵測抽空、`opts=0` → 整題被「有效選項<2」過濾掉（一度整份選擇 15 題消失）。修法：`extract.py` 的 `normalize_pua()` 把 U+F06A..F06E 映回 ①②③④⑤，在 `extract_text_from_pdf` 出口套用。**遇到「選擇題抽到但 opts=0」先疑 PUA 標記**（`[hex(ord(c)) for c in text if ord(c)>=0xE000]` 檢查）。
2. **題組／連連看邊界**：廣興112 選擇跑進「三、題組」造成 mu1-10（金魚/綠繡眼…opts=0）假題。`NON_TARGET_SECTION` 加 `題組` 切界。改 NON_TARGET 務必跑 `pytest`（期中 golden）。
3. **圖片題正確排除**：廣興112 mu13（下圖風向計）、mu14（圖示容器選項）含圖/空選項 → 被既有 filter 正確排除，官方答案讀了但不入庫，正常。
4. **整卷圖片無法處理**：草港112、內安111/112 連題幹都是掃描圖（chars≈0），跳過（家長同意）。
5. **雙欄打散仍是最大弱點**：三民答案卷是非 12→6（漏 1,4,6,7,8,11），抽到的對齊正確、漏的就是沒有。海佃 header 被底線打散＋配合題無邊界，需動通用 regex（高風險），本次暫緩。

### 視覺判讀答案的可靠度
- **整頁縮圖 OCR 不可靠**：兩個獨立 sonnet agent 讀整頁答案卷，分歧極大（連是非 O/X 都不一致）。
- **象限放大（2×2 crop）由主模型親讀才準**：`scripts/crop_pdf.py` 切象限後，紅筆答案清晰可靠。
- **交叉驗證**：`scripts/crosscheck_official_ai.py` 比對視覺官方 vs AI 判答；本批 69 題僅 2 分歧（97% 一致），分歧處官方皆正確（四維是非4「鳥類身體5部位」AI 判錯、廣興112選擇9「一天最高溫」官方答中午12點＝課本答案）。→ **官方優先確實補上 AI 的錯**。

### 併入方式（不動既有已複審資料）
- 既有 `classified_questions_期末.json` 含 review A 人工修正，**不可重跑分類覆蓋**。
- 新卷獨立跑 `classify.py --input/--output`（已加這兩個 CLI 參數）成 `_新增` 檔 → `apply_answer_key.py` 注入官方 → `merge_classified.py` 以正規化題目文字**跨檔去重**後 append（新題文字若已在既有題庫則丟棄，本批 0 重複）→ 照常 `build_review_list.py` + `build_questions.py`。

## 八、第二批 Stage 2：7 份無官方答案卷（2026-06-10）

大墩113/112、文心112/111、路上111（期末3 三段考）、永安113、東光113 → 178 題（去重後）→ +160 入庫。

### 新踩的坑（parser）
1. **阿拉伯數字大題編號**：路上111 的 header 是「1 是非題(每題 4 分…)」「2 選擇題(…)」（前綴還被雙欄打散）。SECTION_PATTERNS 加了**行錨定**替代式 `^[ \t]*\d?[ \t]*是非題[\(（]`（要求題型名後接左括號降誤判，需 re.MULTILINE）。NON_TARGET 同理加 bare `請連連看|做一做`（該卷非目標大題也是阿拉伯編號，靠題幹指示語切界）。
2. **`╳`（U+2573）是非標記**：不在 TF_CHARS，整份是非答案抽空。已加入 TF_CHARS＋normalize_tf_answer。
3. **`(1)(2)(3)(4)` 半形括號選項**：大墩112 #9 單題用此格式（同卷其他題用 ①/○1）。extract_options 加第三 fallback pattern `[\(（]([1-5])[\)）]`（僅前兩種都失敗才試）。
4. **路上111 意外是格式A 內嵌答案**：題目卷前置括號有 ○/╳/①–④ → 16 題官方答案直接到手，不必 AI 補。**拿到「題目卷」先看第一頁有沒有內嵌答案再決定流程**。
5. **classify 過度排除**：燙手縮回／保育獵捕類題被標 none，但主題庫先例（民權 tr#3、三民 mu#5）同概念都是 unit3。**none 清單要過一遍、用主題庫先例校正**（grep 同關鍵詞看既有 unit/subtopic）。

### 盲審 review B（148 題、6 agent 平行、95.3% 一致）
- 工具化：`scripts/build_review_blind.py` 產盲版；agent 各審一段 index（自己 Read 檔，args 只傳範圍）。
- 7 分歧定奪：5 題改盲審答案（魚鱗雲=晴天、專線=166/167 等**主題庫先例可直接佐證**）、百葉箱題 websearch 後維持 AI 答案、東光 mu4 正解不在選項中→剔除。
- 經驗：**分歧題先 grep 主題庫同概念題**，先例往往直接給答案；真沒把握再 websearch 或問家長。
